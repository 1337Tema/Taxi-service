Задания для System Design

1. Сервис заказа такси (аналог Uber, Lyft, Яндекс.Такси)
Предложите свой дизайн для системы глобального сервиса заказа такси.
 Система представлена мобильным приложением и веб-интерфейсом, доступна на рынке практически всех стран мира, с ±100M MAU и ±10M DAU.
Уровни сложности выполнения задачи
Уровень первый, попроще:
Описать и рассчитать MVP с самым ключевым функционалом:
Регистрация и авторизация пользователей.
Поиск ближайшего водителя и заказ поездки.
Просмотр истории поездок.
Реализация основных API для взаимодействия пассажиров и водителей.
Хранение данных о профилях пользователей, транспортных средствах и поездках.
Продумать, как будет работать уведомление о статусе поездки (ожидание водителя, начало поездки, завершение).
Уровень второй, посложнее:
Учитывать геолокацию и поиск водителей на основе их положения. Продумать:
Как обрабатывать миллионы запросов на поиск водителей.
Подходы к шардированию данных и балансировке нагрузки.
Реализацию алгоритма распределения поездок с минимальной задержкой.
Добавить возможность расчета стоимости поездки в реальном времени.
Реализовать функциональность для поддержки встраиваемых карт и маршрутов.
Уровень третий, хардкор:
Система должна обрабатывать и хранить 1M events/sec, включая действия пользователей, маршруты и телеметрию автомобилей.
Предложить ETL-конвейер для обработки данных о поездках, водителях и пассажирах.
Разработать несколько ML-компонент:
Прогнозирование времени прибытия водителя.
Динамическое ценообразование на основе спроса и предложения.
Оптимизация распределения водителей по районам.
